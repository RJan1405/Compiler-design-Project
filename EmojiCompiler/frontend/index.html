The 5 Stages of Compilation

01

Lexical Analysis

The Lexer converts raw emoji strings into Tokens. It uses Regex to identify 🙂 as START, 🆕 as NEW_VAR, and handles multi-digit numbers.

02

Syntax Analysis

The Parser builds an Abstract Syntax Tree (AST). It ensures that expressions like a ➕ b have two operands and correctly handles the ❓ ... ➡ structure.

03

Semantic Analysis

This stage checks the "meaning". It builds a Symbol Table to ensure variables are declared before use and prevents duplicate declarations of the same emoji-variable.

04

Intermediate Code

Generates Three-Address Code (TAC). This simplifies complex logical chains into basic operations, making it easy to translate the logic to any target language, not just Python.

05

Python Generation

The final stage translates the tree into executable Python. It carefully manages indents (using a global level tracker) so that IF/WHILE blocks work perfectly.

The Engineering Blueprint

🐍

lexer.py

Defines the Token class and Lexer. It is the entry gate for all emoji input.

🐍

parser.py

The logic engine. Implements recursive descent to map tokens to AST nodes like IfNode or BinOpNode.

🐍

semantic.py

The validator. It traverses the AST and catches errors like "Variable Not Found" using its internal symbol dictionary.
🐍

codegen.py

The translator. It has an indent_level tracker that ensures every ❓ and 🔄 block is indented correctly in Python.
🐍

app.py

The API bridge. It connects this web interface to the compiler logic and captures the final terminal output buffer.
